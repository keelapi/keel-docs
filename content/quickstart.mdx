# Quickstart

Get your first permit decision in 60 seconds

This quickstart shows the shortest 3-step path through the live Keel API using copy-paste-friendly requests.

## 1. Create an API key

Create a project API key in Keel, then use it in the standard Bearer header:

```text
Authorization: Bearer keel_sk_your_key_here
```

## 2. Send a permit request

```bash
curl -sS https://api.keelapi.com/v1/permits \
  -H "Authorization: Bearer keel_sk_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "project_id": "<project_uuid>",
    "idempotency_key": "quickstart-permit-001",
    "subject": {
      "type": "user",
      "id": "usr_demo"
    },
    "action": {
      "name": "ai.generate.summary"
    },
    "resource": {
      "type": "request",
      "id": "req_quickstart_001",
      "attributes": {
        "provider": "openai",
        "model": "gpt-4o-mini",
        "operation": "generate.text",
        "modality": "text",
        "execution_mode": "sync",
        "estimated_input_tokens": 120,
        "estimated_output_tokens": 80,
        "max_output_tokens_requested": 120
      }
    }
  }'
```

Look for a decision like `allow` and keep the returned permit metadata for audit and usage reporting.

## 3. Execute a model

```bash
curl -sS https://api.keelapi.com/v1/executions \
  -H "Authorization: Bearer keel_sk_your_key_here" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: quickstart-exec-001" \
  -d '{
    "operation": "generate.text",
    "messages": [
      {"role": "user", "content": "Summarize why Keel evaluated this request first."}
    ],
    "routing": {
      "provider": "openai",
      "model": "gpt-4o-mini"
    },
    "parameters": {
      "max_output_tokens": 80
    }
  }'
```

That request gives you a governed model response plus routing, usage, and permit metadata in one API call.
