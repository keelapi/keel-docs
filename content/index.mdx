# Keel Docs

Keel is a permit-driven governance and execution control layer in front of provider APIs. It gives teams a single place to enforce decisioning, routing, budget controls, and auditability before AI traffic reaches a model provider.

## What Keel covers

- permit decisions before provider dispatch
- provider-neutral and provider-native execution paths
- routing, fallback, and budget controls
- prompt inspection on supported execution routes
- persisted audit records, timelines, and compliance exports

## Recommended path

1. Read [Overview](/docs/overview) to understand the product boundary.
2. Run [Quickstart](/docs/quickstart) to get a governed request through the stack.
