# Platform Surfaces

This page tracks the implemented route taxonomy for Keel's mounted AI runtime, request lifecycle, compatibility, and operator surfaces.

## Surface taxonomy

Use this page as the mounted-surface map for beta:

- decision surfaces: permit creation and permit readback and audit routes
- execution surfaces: unified execution, provider-native proxy execution, and async job submission and status
- lifecycle surfaces: request timeline replay, governance event readback, event streaming, and compliance export flows
- governance and telemetry surfaces: metrics, provider diagnostics, billing webhook ingestion, and public provider health status
- management and operator surfaces: project, policy, provider-key, dashboard, organization, and control-plane routes

## Beta support matrix

| Surface family | Beta status | Notes |
| --- | --- | --- |
| Permit decision contract | Stable | `POST /v1/permits` remains the canonical public decision seam. |
| Unified execution | Stable | `POST /v1/execute` and `POST /v1/executions` are active public execution surfaces with different request contracts. |
| Proxy execution | Limited | Capability coverage differs by provider. Public routing exists only on OpenAI and Anthropic. Public streaming is effectively limited to the current OpenAI path. |
| Async jobs and lifecycle replay | Stable | Public async job submission, status, and request timeline replay are mounted. |
| Governance events and compliance exports | Stable | Public project-scoped readback and export surfaces are mounted. |
| Dashboard and control-plane surfaces | Internal only | Operator- and UI-focused surfaces, not canonical public runtime APIs. |
| Realtime session transport | Internal only | Persistence scaffolding exists, but there is no public session-start route. |

## Official public runtime and lifecycle

| Route | Notes |
| --- | --- |
| `POST /v1/execute` | Unified execution endpoint backed by the normalized execution pipeline. Accepts provider-shaped `input`, infers operation from payload, returns the normalized execution envelope, and adds a `resolved` block. |
| `GET /v1/permits` | Permit audit and readback. |
| `GET /v1/permits/{permit_id}` | Permit audit and readback. |
| `GET /v1/permits/export` | Project-scoped audit export. |
| `POST /v1/permits/{permit_id}/usage` | Permit-first usage reporting. Public completed reports still require positive billed cost. |
| `POST /v1/executions` | Provider-neutral execution route. Supports the current public sync subset plus the current OpenAI streaming path, and returns a normalized execution envelope. |
| `GET /v1/executions/{request_id}` | Public execution inspector and readback route for the authenticated project. |
| `POST /v1/proxy/openai` | Provider-specific execution route. Supports route-specific streaming plus payload-inferred text, image-understanding, image-generation or edit, and audio operations. Public `routing` supported. |

## Current boundaries

- `/v1/permits` remains the canonical public decision seam
- `/v1/executions` and `/v1/execute` are both active public execution routes, but they do not have the same request contract
- `/v1/executions` is provider-neutral at the request boundary, but it is still a narrower contract than the provider-native proxy routes
- `/v1/execute` accepts a provider-shaped `input` object and returns the normalized execution envelope plus a `resolved` block
- OpenAI is the only public proxy route with implemented route-specific streaming behavior
- public routing controls are currently exposed on `/v1/executions`, `/v1/proxy/openai`, and `/v1/proxy/anthropic`
- public cross-provider fallback is explicit, non-stream, and limited to the implemented OpenAI and Anthropic text-translation subset
- project-scoped provider keys are shared execution infrastructure across `/v1/executions`, `/v1/execute`, `/v1/proxy/*`, async jobs, and dashboard gateway test execution
- async jobs are public; realtime sessions are persisted internally for lifecycle replay, but there is no public realtime session API
