# API Reference

This section is the entry point for the first authored developer-facing API docs imported into this Nextra shell. It does not replace a generated OpenAPI reference yet, but it now covers the main public execution contracts and the error and retry behavior you need to integrate against them.

## Start here

- [Permits](/permits) for the canonical decision contract
- [Executions](/executions) for the provider-neutral execution route
- [Execute](/execute) for the lower-level provider-shaped execution route
- [Proxy Execution](/proxy-execution) for the provider-native runtime routes
- [Idempotency](/idempotency) for retry and replay semantics across public surfaces
- [Errors](/errors) for the stable minimum error contracts

## Current state

Today, the Keel backend exposes:

- authored API docs in this site for the first public developer-facing routes
- a FastAPI-generated OpenAPI schema in non-production environments
- public runtime routes under `https://api.keelapi.com`

## Planned follow-up

The next iteration should add:

- generated OpenAPI pages or an embedded reference viewer
- stable links for `openapi.json` and hosted interactive docs
- deeper authored API pages for usage reporting and route-specific readback flows
